# 1  项目概况

## 1.1   项目背景

随着业务的发展，为保证app后续可以更加契合业务的发展，同时为后续更好的维护C端触点的初衷，以及实现敏捷的方式进行迭代升级，所以需进行恒驰app转自研。

## 1.2   项目目标

本次主要针对app一期预售的内容进行产品设计和研发落地，其主要包含的内容如下：

### 2.1.13登录/注册

移动端登录页集成集团统一用户平台的SDK，从移动端进入时跳转统一用户页面，默认为验证码登录的方式，新用户首次登录时需要先注册，可切换为密码登录。

汽车集团的用户中心为目前自研项目新建的用户中心，主要为汽车集团提供统一的用户服务。

功能摘要

1）注册登录：移动端用户登录系统，包括如下内容：设置头像、设置新密码、注销用户等；

2）个人中心：我的频道的个人中心; 

3）具体功能以高科技最新版本的SDK为准；

4）新用户注册时，用户创建通过SDK将用户创建在集团的统一用户中心；但APP及server需要将该用户信息同步至汽车集团的用户中心，为其创建对应的汽车集团的用户唯一id；

5）用户在登录成功后，也需要将用户信息同步至汽车集团的用户中心，若该用户在汽车集团未创建过用户，则需在汽车集团创建该用户信息；

6）APP的登录态，以集团用户中心的（或汽车集团，以架构设计为准）的用户登录态为准，该登录态的有效期为14个自然日（暂停）；每次APP启动成功则按照当前时间节点自动延期14天用户的登录态；需注意，商城的登录态是以汽车集团的用户ID进行用户身份标识；

7）用户在APP端退出登录后，则清除用户登录态，且只清除APP对应的登录态；

8）APP修改手机号后，则需登出APP；

| **1**          | 圈子管理列表               | 1、展示列表，展示以下数据列；  圈子名称、圈子摘要、状态，支持对每条话题数据进行操作：编辑、置顶/取消置顶、启用/停用；一页显示的数据量，支持下拉定义；  2、列表查询说明；  圈子名称、状态搜索；  圈子名称支持模糊搜索；  3、操作按钮操作定义；  新增：点击新增按钮，弹出新增界面；  编辑：点击编辑按钮，弹出当前信息的编辑界面；  启用/停用：对圈子进行启用/停用操作，停用的圈子，在APP不显示；  置顶/取消置顶：已启用状态下的圈子，可以进行置顶/取消置顶操作，置顶：前端优先展示置顶圈子，多个置顶，按置顶时间降序展示； |
| **2**          | 圈子管理编辑/新增          | 编辑/新增界面包含以下内容：  1、圈子名称：必填，最多支持输入30个字符；  2、圈子标签：必填，可输入多个，通过“,”进行不同标签分割，最多支持输入200个字符；  3、圈子状态：分启用/停用，必填；停用：APP不显示；  4、圈子摘要：必填，最多支持输入200个字符；  5、封面图：仅支持图片格式：jpg/png，文件大小：建议100K以下，大小：800 x 800，图片必填； |
| **3**          | 话题管理列表               | 展示列表，展示以下数据列；  1、话题名称、是否热门、话题摘要、状态，支持对每条话题数据进行操作：编辑、启用/停用；一页显示的数据量，支持下拉定义；  2、列表查询说明；  话题名称、是否热门、状态搜索；  圈子名称支持模糊搜索；  3、操作按钮操作定义；  新增：点击新增按钮，弹出新增界面；  编辑：点击编辑按钮，弹出当前信息的编辑界面；  启用/停用：对话题进行启用/停用操作，停用的话题，在APP不显示； |
| **4**          | 话题管理编辑/新增          | 编辑/新增界面包含以下内容：  1、话题名称：必填，最多支持输入20个字符；  2、是否热门：分是/否，必选，热门的话题，APP显示热门的图标，前端优先展示热门的话题；  3、话题状态：分启用/停用，必填；停用：APP不显示；  4、话题摘要：必填，最多支持输入200个字符；  5、封面图：仅支持图片格式：jpg/png，文件大小：建议100K以下，大小：800 x 800，图片必填； |
| **5**          | 提问管理列表               | 展示列表，展示以下数据列；  1、问题标题、问题内容、提问类型、提问人、提问时间、状态，支持对话题数据进行操作：置顶/取消置顶、审核、详情、删除；一页显示的数据量，支持下拉定义；  2、列表查询说明；  问题标题、问题内容、提问人、提问类型、是否置顶、状态、提问时间搜索；  问题标题、问题内容、提问人、支持模糊搜索；  3、操作按钮操作定义；  置顶/取消置顶：已通过状态下的提问，可以进行置顶/取消置顶操作，置顶：前端优先展示置顶提问，多个置顶，按置顶时间降序展示；  审核：待审核状态下的提问才有审核操作入口，APP用户发布的提问需要先审核，审核通过才能显示在APP，审核不通过，不显示在APP；  详情：点击详情按钮，跳转对应提问的详情页面；  删除：非删除状态的提问才有删除入口，为逻辑删除；  批量删除：批量逻辑删除提问； |
| **6**          | 提问审核                   | 审核界面包含以下内容：  1、问题标题：提问的标题，只显示；  2、问题内容：提问的内容，只显示；  3、提问类型：提问的类型，只显示；  4、提问人：问题的发布者，只显示；  5、提问时间、提问图片：只显示；  6、状态：提问审核状态，可选审核通过和不审核通过，必选，审核不通过需要选择原因，审核不通过的提问，APP不显示，审核通过，APP显示；  7、原因：审核不通过的提问，需要选择原因：“包含敏感信息”或“内容不合法”，必选；  备注：审核备注信息，必填，最多支持输入200个字符； |
| **7**          | 提问详情                   | 详情界面包含以下内容：  1、问题标题、问题内容、提问类型、提问人、提问时间、图片、状态、原因、备注； |
| **8**          | 问答管理列表               | 展示列表，展示以下数据列；  1、问题标题、问题内容、回答内容、回答人、回答时间、状态，支持对话题数据进行操作：审核、详情、删除；一页显示的数据量，支持下拉定义；  2、列表查询说明；  问题标题、回答内容、回答人、评论类型、状态、回答时间搜索；  问题标题、回答内容、回答人支持模糊搜索；  3、操作按钮操作定义；  审核：待审核状态下的问答才有审核操作入口，APP用户发布的问答需要先审核，审核通过才能显示在APP，审核不通过，不显示在APP；  详情：点击详情按钮，跳转对应问答的详情页面；  删除：非删除状态的问答才有删除入口，为逻辑删除；  批量删除：批量逻辑删除问答； |
| **9**          | 问答审核                   | 审核界面包含以下内容：  1、提问的内容显示：  问题标题：提问的标题；  问题内容：提问的内容；  问题内容：提问的图片；  2、回答的内容显示：  回答内容：回答的内容，只显示；  回答人、回答时间、图片：只显示；  状态：回答的审核状态，可选审核通过和不审核通过，必选，审核不通过需要选择原因，审核不通过的回答，APP不显示，审核通过，APP显示；  3、原因：审核不通过的回答，需要选择原因：“包含敏感信息”或“内容不合法”，必选；  备注：审核备注信息，必填，最多支持输入200个字符； |
| **10**         | 问答详情                   | 详情界面包含以下内容：  1、提问的内容显示：  问题标题、问题内容、图片；  2、回答的内容显示：  回答内容、回答人、回答时间、图片、状态、原因、备注。 |
| **11**         | 动态管理列表               | 展示列表，展示以下数据列；  1、标题、内容、所属圈子、所属话题、发布人、发布时间、状态，支持对话题数据进行操作：置顶/取消置顶、审核、详情、删除；一页显示的数据量，支持下拉定义；  2、列表查询说明；  标题、内容、发布人、所属话题、所属圈子、是否置顶、状态、发布时间搜索；  标题、内容、发布人、所属话题、所属圈子支持模糊搜索；  3、操作按钮操作定义；  新增：点击新增按钮，弹出新增界面；  编辑：点击编辑按钮，弹出当前信息的编辑界面；  置顶/取消置顶：已通过状态下的动态，可以进行置顶/取消置顶操作，置顶：前端优先展示置顶动态，多个置顶，按置顶时间降序展示；  审核：待审核状态下的动态才有审核操作入口，APP用户发布的动态需要先审核，审核通过才能显示在APP，审核不通过，不显示在APP；  详情：点击详情按钮，跳转对应动态的详情页面；  删除：非删除状态的动态才有删除入口，为逻辑删除；  批量删除：批量逻辑删除动态； |
| **12**         | 动态审核                   | 审核界面包含以下内容：  1、标题：动态标题，只显示；  2、内容：动态内容，只显示；  3、所属圈子：动态所属圈子，只显示；  4、所属话题：动态所属话题，只显示；  5、发布人、发布时间、动态图片：只显示；  6、状态：动态审核状态，可选审核通过和不审核通过，必选，审核不通过需要选择原因，审核不通过的动态，APP不显示，审核通过，APP显示；  7、原因：审核不通过的动态，需要选择原因：“包含敏感信息”或“内容不合法”，必选；  备注：审核备注信息，必填，最多支持输入200个字符； |
| **13**         | 动态详情                   | 详情界面包含以下内容：  1、动态标题、动态内容、所属圈子、所属话题、发布人、状态、原因、备注、评论数、点赞数、转发数、收藏数； |
| **14**         | 评论管理列表               | 展示列表，展示以下数据列；  1、动态标题、评论类型、评论内容、发布人、被评论用户、被评论内容、评论时间、状态，支持对话题数据进行操作：审核、详情、删除；一页显示的数据量，支持下拉定义；  评论类型：分评论楼主和评论回复；评论楼主：对动态的评论；评论回复：对动态评论的回复。  被评论用户：如果是评论楼主，被评论用户是指动态的发布人，如果是评论回复，被评论用户是指评论的发布人；  被评论内容：如果是评论楼主，被评论内容指是动态的内容，如果是评论回复，被评论内容是指评论的内容；  2、列表查询说明；  动态标题、评论内容、发布人、评论类型、状态、评论时间搜索；  动态标题、评论内容、发布人支持模糊搜索；  3、操作按钮操作定义；  审核：待审核状态下的评论/回复才有审核操作入口，APP用户发布的评论/回复需要先审核，审核通过才能显示在APP，审核不通过，不显示在APP；  详情：点击详情按钮，跳转对应评论/回复的详情页面；  删除：非删除状态的动态才有删除入口，为逻辑删除；  批量删除：批量逻辑删除评论/回复； |
| **15**         | 评论审核                   | 审核界面包含以下内容：  1、被评论的内容显示：  动态标题：被评论的动态标题；  被评论用户：被评论的内容的发布人；  评论内容：如果是评论楼主，评论内容指是动态的内容，如果是评论回复，评论内容是指评论的内容；  2、评论的内容显示：  评论内容：评论的内容，只显示；  发布人、评论时间、图片：只显示；  状态：评论审核状态，可选审核通过和不审核通过，必选，审核不通过需要选择原因，审核不通过的评论，APP不显示，审核通过，APP显示；  3、原因：审核不通过的评论，需要选择原因：“包含敏感信息”或“内容不合法”，必选；  备注：审核备注信息，必填，最多支持输入200个字符； |
| **16**         | 评论详情                   | 详情界面包含以下内容：  1、被评论的内容显示：  动态标题、被评论用户、评论内容：图片；  2、评论的内容显示：  评论内容、发布人、评论时间、图片、状态、原因、备注、点赞数。 |
